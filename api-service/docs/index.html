Authentication

All API requests require authentication using an API key passed in the request headers.

Getting an API Key

Contact your administrator to obtain an API key for your application.

Using Your API Key

Include your API key in the X-API-Key header with every request:

curl -X GET "http://localhost:3000/v1/platforms" \
  -H "X-API-Key: your-api-key-here"
Security Note: Never expose your API key in client-side code or public repositories. Always use environment variables or secure configuration management.

Quick Start

1. Install SDK (Optional)

# JavaScript/Node.js
npm install ish-orchestrator-sdk

# Python
pip install ish-orchestrator-sdk

2. Make Your First Request

// JavaScript
const client = new ISHOrchestratorClient({
  apiKey: 'your-api-key',
  baseUrl: 'http://localhost:3000'
});

const result = await client.query({
  query: 'Explain quantum computing',
  platform: 'auto'
});

console.log(result.data.result.response);

3. Explore the API

Visit the interactive API documentation to explore all endpoints and try them directly in your browser.

API Endpoints

Queries

POST /v1/query

Submit a new query for processing

GET /v1/query/:id

Retrieve query results by ID

GET /v1/query/:id/stream

Stream query results using Server-Sent Events

Platforms

GET /v1/platforms

List all available AI platforms

GET /v1/platforms/:name/status

Get platform health status

Batch Processing

POST /v1/batch

Submit multiple queries for batch processing

GET /v1/batch/:id

Get batch processing results

Comparison

POST /v1/compare

Compare responses across multiple platforms

Analytics

GET /v1/stats

Get usage statistics and analytics

GET /v1/stats/export

Export analytics data as CSV

Examples

Example 1: Simple Query

const result = await client.query({
  query: 'What is the meaning of life?',
  platform: 'auto',
  temperature: 0.7
});

console.log(result.data.result.response);

Example 2: Batch Processing

const queries = [
  { id: 'q1', query: 'What is AI?', platform: 'claude' },
  { id: 'q2', query: 'Explain blockchain', platform: 'gpt' },
  { id: 'q3', query: 'What is quantum computing?', platform: 'gemini' }
];

const result = await client.batch(queries);

result.data.queries.forEach(q => {
  console.log(`${q.id}: ${q.result.response}`);
});

Example 3: Platform Comparison

const result = await client.compare(
  'Write a haiku about technology',
  ['claude', 'gpt', 'gemini']
);

result.data.results.forEach(r => {
  console.log(`\n${r.platform}:\n${r.result.response}`);
});

console.log('\nBest for speed:', result.data.summary.fastest.platform);

Rate Limits

To ensure fair usage and system stability, API requests are rate-limited per API key.

Limit Type Value Window
Request Limit 100 requests 15 minutes
Batch Size 10 queries Per batch
Compare Platforms 4 platforms Per request

Rate Limit Headers

Every response includes rate limit information in the headers:

  • X-RateLimit-Limit: Maximum requests per window
  • X-RateLimit-Remaining: Remaining requests in current window
  • X-RateLimit-Reset: Time when the limit resets
Tip: Monitor these headers to avoid hitting rate limits. Implement exponential backoff for failed requests.
